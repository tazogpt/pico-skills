## Update Policy

포함한다:

- 시스템 한 줄 설명
- 모듈과 디렉터리 구조
- 계층과 의존 방향
- 주요 도메인 경계
- 핵심 API·DB·권한·라우팅 계약
- 공통 구현 패턴
- 금지된 의존성 또는 폐기된 방식
- 중요한 아키텍처 결정과 근거
- 현재 알려진 구조적 제약

포함하지 않는다:

- 개별 태스크의 구현 순서
- 임시 작업 메모
- 전체 코드 목록
- 전체 변경 이력
- 특정 에이전트 프롬프트
- 오래된 계획 또는 폐기된 가정

수정 권한:

- 리뷰어와 임플리먼터는 직접 수정하지 않는다.
- 불일치와 갱신 후보를 아키텍트에게 보고한다.
- 아키텍트만 갱신 필요 여부와 범위를 결정한다.
- 구현 리뷰 PASS 후 필요한 경우 지정된 범위만 갱신한다.
- 단순 내부 구현 변경은 기록하지 않는다.

완료 게이트:

```text
ARCHI_CHECK
- update not required
```

또는:

```text
ARCHI_CHECK
- updated
- changed facts: ...
```

갱신이 필요한데 반영되지 않았다면 워크플로를 완료하지 않는다.
